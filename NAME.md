# Naming the Project

## Context

The project is currently called `email` -- too generic to publish. We need a memorable name for:
- GitHub repo and website
- The binary (`cargo install`)
- A short alias for daily use (TUI launch + CLI commands)
- AI agent invocation

The project is a **non-commercial, open-source Rust TUI/CLI email client**. Emails are Markdown files with YAML frontmatter. The tone should reflect that it's a fun side project, not enterprise software.

---

## Current Line-up

### mailypoppins

- Alias: `mp`
- Tone: playful, memorable, conversation starter
- Googleability: perfect -- zero collisions
- Domain: `mailypoppins.dev` certainly available
- Pros: instantly memorable, makes people smile, signals "fun side project," the pun works (Mary Poppins delivers things, keeps everything tidy)
- Cons: long to type in full, but `mp` solves it

### lazymail

- Alias: `lm`
- Tone: serious tool, friendly name
- Googleability: decent, some noise
- Domain: `lazymail.dev` likely available
- Pros: rides the `lazy*` naming wave (lazygit, lazydocker), immediately signals "nice TUI for X"
- Cons: "lazy" slightly undersells the Markdown-first philosophy; more conventional

---

## Rejected Ideas

| Name | Why rejected |
|------|-------------|
| **nib** | Too obscure -- non-native speakers won't know the word |
| **supermail** | Generic, no personality |
| **easymail** | Sounds like a 2005 webmail startup |
| **quill** | Nice but conventional, no strong differentiator |
| **herald** | Cool but no immediate email association for most people |
| **dispatch** | Long to type, generic verb |
| **inkwell** | Long, abstract |
| **envelop** | Too subtle a pun |
